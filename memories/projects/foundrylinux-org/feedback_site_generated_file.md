Never edit `site/index.html` directly. It is generated by `task site-build` (`node scripts/ssr-render.js`) and any direct edits will be overwritten.

**Why:** The HTML is SSR-rendered from `site/app.jsx` + `site/sections.jsx`. Inline scripts and page structure live in `scripts/ssr-render.js`.

**How to apply:** For copy/markup changes → edit the relevant `site/*.jsx` file. For injected scripts or page shell changes → edit `scripts/ssr-render.js`. Always run `task site-build` after and commit the regenerated `site/index.html` alongside the source.
