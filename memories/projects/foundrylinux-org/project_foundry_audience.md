Foundry Linux is **not** targeted at traditional Linux users. It targets **game developers who may never have used Linux before** and don't want to think about it — Linux is just the platform they're building games on.

**Why:** These are people coming from Windows/Mac game dev backgrounds. They don't have Linux community preconceptions, don't care about distro philosophy, GPG trust chains, package manager debates, etc. Linux is just a tool. Foundry Linux's job is to make that tool invisible.

**How to apply:**
- Never frame UX, hosting, or download experience decisions in terms of "what Linux users expect" — frame them in terms of "what a game developer downloading a tool expects"
- Download experience should feel like downloading Unity or Godot — click button, get file, run installer
- Documentation and UI copy should not assume Linux familiarity
- SourceForge's reputation risk for our audience is **confusing ad-cluttered download pages with fake Download buttons**, not the 2015 adware incident (our users don't know about that)
- Internet Archive's relevance: our users don't know it as a software host (they know WayBack Machine), but this doesn't matter because they never see the backend URL — they click "Download" on foundrylinux.org
- Torrent links ARE relevant — gamers are very familiar with torrents for large downloads; a 10 GB ISO is exactly the use case
